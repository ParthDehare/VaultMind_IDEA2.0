# VaultMind 2.0

## Problem Statement
This project addresses PS1: AI-Driven Early Warning System for Internal & Privileged User Fraud (Union Bank of India) by analyzing internal bank employee behavior in real-time, correlating behavioral signals across multiple layers, and detecting anomalous activity before financial loss occurs.

## How to Run Locally
Step-by-step instructions:
1. Clone the repo: `git clone https://github.com/ParthDehare/VaultMind_IDEA2.0`
2. Navigate to the root directory.
3. Run the application using Docker Compose:
   ```bash
   docker-compose up -d --build
   ```
4. Open your browser and navigate to: http://localhost:5173

## Project Structure
Briefly describe what each folder/file does:
/server — FastAPI backend housing the Data Fusion Engine, ML models/agents, and auth/utility API routes.
/client — React.js (Vite) frontend containing the 7-screen interactive early-warning command center.
/docker-compose.yml — Defines and runs the multi-container system (Zookeeper, Kafka, Redis, backend, frontend, producer).
/Dockerfile.api — Docker configuration for compiling and spinning up the Python backend/producer containers.
/Dockerfile.web — Docker configuration for building and serving the React frontend web application.
/nginx.conf — Reverse proxy server config to direct network requests cleanly to frontend and backend services.
/START_ALL.bat — One-click launcher script for starting services locally on Windows without Docker.

## Dataset
Describe the data used:
All data is 100% synthetic, generated by our team. It simulates live core banking internal transactions, system logs, and privileged employee access patterns across multiple bank roles. This high-fidelity simulated stream is continuously pushed through Apache Kafka to test the system's ability to trace, score, and alert anomalous behaviors in real-time without using any real banking or customer data.

## Known Limitations
Be honest — evaluators respect honesty:
• Trained only on synthetic data; would need real transaction data and verified bank logs for production use.
• Dependent on continuous retraining pipelines to prevent model drift as long-term bank operations and employee behavior shift.
• Real-time high-throughput streaming with multiple deep learning agents places a high CPU/Memory demand on local resources.
